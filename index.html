Posted on 2026-06-10

Git命令汇总

1. 历史查看与调试

1.1. git shortlog：提交汇总，按作者分组统计提交次数与提交说 明

1.2. git blame：追溯作者，逐行显示代码最后修改人和提交信息

1.3. git bisect：二分查找，通过二分法快速定位引入 bug 的提交

1.4. git reflog：引用日志，记录所有 HEAD 变动用于误操作恢复

1.5. git whatchanged：查看变更，显示每次提交涉及的具体文件 改动

1.6. git rev-list：提交列表，按条件筛选并列出匹配的提交记录

2. 补丁与归档工具

2.1. git format-patch：生成补丁，将提交导出为可邮件发送的补 丁文件

2.2. git apply：应用补丁，将 diff 补丁文件中的改动应用到工作 区

2.3. git am：应用补丁，将邮件格式的补丁文件应用到仓库

2.4. git archive：打包归档，将指定提交导出为 tar 或 zip 压缩包

2.5. git bundle：打包传输，将分支打包为单个文件方便离线传递

3. 仓库维护与性能优化

3.1. git gc：垃圾回收，清理冗余对象压缩仓库优化存储空间

3.2. git maintenance：维护任务，定期执行 gc 等优化保持仓库 性能健康

3.3. git count-objects：对象统计，统计仓库中松散对象和包文 件的数量

3.4. git repack：重新打包，将松散对象压缩为高效的包文件

3.5. git prune：修剪对象，删除不再被任何引用指向的孤立对象

3.6. git fsck：仓库检查，扫描对象数据库并报告损坏或丢失

3.7. git sparse-checkout：稀疏检出，仅拉取指定目录大幅节省 大仓库空间

4. 高级对象与引用操作

4.1. git rev-parse：解析引用，将分支名或标签转换为完整 SHA 哈希值

4.2. git symbolic-ref：符号引用，读取或修改 HEAD 等符号引用 的指向目标

4.3. git cat-file：查看对象，查看 Git 内部对象的类型、大小和内 容

4.4. git replace：对象替换，用新对象替换历史记录中的旧对象

4.5. git notes：添加注释，为已有提交附加额外说明不影响 SHA

4.6. git update-index：强制更新，手动修改暂存区文件状态或跳 过检测

4.7. git filter-branch：重写历史，批量修改提交历史如删除大文 件等

4.8. git verify-commit：验证签名，校验 GPG 签名提交的完整性 与真实性

4.9. git verify-tag：验证标签，校验标签附带的 GPG 签名是否有 效

4.10. git describe：版本描述，基于最近的标签生成人类可读版 本号

5. 图形化与调试工具

5.1. git gui：图形界面，启动 Git 自带的可视化操作客户端

5.2. gitk：图形历史，启动 Tcl/Tk 图形界面可视化查看提交

5.3. git instaweb：即时网页，启动 gitweb 服务器图形化浏览仓 库历史

5.4. git difftool：差异对比，使用配置的外部工具查看文件差异

5.5. git merge-file：文件合并，对单个文件执行三路合并生成结 果

5.6. git diagnose：诊断工具，分析仓库状态并输出详细的诊断日 志

5.7. git bugreport：Bug 报告，收集系统和配置信息生成标准化 报告

6. 其他高级功能

6.1. git grep：仓库搜索，在版本库所有文件中快速查找内容

6.2. git worktree：工作树管理，在同一仓库创建多个并行工作目 录

6.3. git submodule：子模块管理，嵌套引用外部仓库作为项目依 赖

6.4. git credential：凭证管理，配置密码缓存或调用外部凭证助 手

6.5. git for-each-ref：遍历引用，对所有分支和标签执行指定格 式化输出

7. 基础配置与仓库初始化

7.1. git config：配置管理，设置用户名、邮箱及仓库行为参数

7.2. git init：初始化仓库，在当前目录创建一个新的 Git 仓库

7.3. git clone：克隆仓库，从远程地址复制完整项目到本地

8. 文件与提交操作

8.1. git add：添加文件，将工作区修改放入暂存区准备提交

8.2. git commit：提交更改，将暂存区内容保存到本地版本库

8.3. git status：查看状态，显示工作区与暂存区的文件变更情况

8.4. git diff：查看差异，比较工作区、暂存区或版本间改动

8.5. git log：查看历史，显示提交记录、作者与时间等详情

8.6. git show：查看详情，显示某次提交的改动内容与元数据

8.7. git rm：删除文件，从工作区和暂存区同时移除指定文件

8.8. git mv：重命名文件，移动或改名同时保留版本追踪记录

8.9. git clean：清理文件，删除工作区中未被 Git 跟踪的文件

8.10. git restore：恢复文件，撤销工作区或暂存区中的文件修改

8.11. git reset：回退版本，撤销提交或将暂存区文件移回工作区

8.12. git stash：暂存更改，临时保存未提交的修改以便切换分支

8.13. git revert：撤销提交，生成新提交来反向抵消指定改动

9. 分支与合并管理

9.1. git branch：分支管理，查看、创建或删除代码开发分支

9.2. git checkout / switch：切换分支，切换到指定分支进行代码 开发

9.3. git merge：合并分支，将指定分支的代码合并到当前分支

9.4. git mergetool：冲突解决，调用外部可视化工具处理合并冲 突

9.5. git rerere：冲突复用，自动记录合并冲突的解决方案供后续 复用

9.6. git range-diff：范围对比，比较两个提交范围之间的差异变 化

9.7. git cherry-pick：挑选提交，将指定提交的改动应用到当前分 支

9.8. git rebase：变基重写，将当前分支提交移到目标分支顶端

9.9. git show-branch：分支拓扑，可视化展示多个分支的提交关 系图

10. 远程仓库交互

10.1. git remote：远程管理，查看、添加或删除远程仓库地址配 置

10.2. git fetch：获取更新，从远程仓库下载最新提交但不合并

10.3. git pull：拉取更新，从远程仓库获取最新代码并合并

10.4. git push：推送代码，将本地提交上传到远程仓库分支

10.5. git ls-remote：查看远程，列出远程仓库的所有引用与分支 信息

10.6. git request-pull：请求合并，生成拉取摘要要发送给项目维 护者

10.7. git send-email：发送补丁，通过 SMTP 将补丁文件以邮件 形式发送

1. Mem Reduct

工具简介

全球高人气开源轻量内存清理工具，安装包体积小巧，提供便携版，兼容Windows7至Win11全系统。依托Windows原生底层API，可深度清理8大内存区域，清理效果出众，是硬核玩家的常用选择。

✅ 优点

  • 体积小巧，自身资源占用极低，支持后台静默运行
  • 支持定时、内存阈值两种自动清理模式，自定义选项丰富
  • 实时监控内存状态，托盘图标可直观区分内存负载情况
  • 开源免费且带有安全签名，全程无广告、无捆绑程序
  • 支持命令行操作，可搭配脚本实现自动化运维使用

❌ 缺点

  • 调用系统未公开API，容易被杀毒软件误报风险
  • 开启激进清理模式时，可能造成系统短暂卡顿
  • 没有中文操作界面，对零基础新手存在使用门槛

2. Windows Memory Cleaner

🔗 官网：https://winmemorycleaner.com/

🔗 GitHub地址：https://github.com/IgorMundstein/WinMemoryCleaner

工具简介

开源免费的全能内存清理工具，累计下载量超470万，适配Windows XP到Win11及各类服务器系统，可精准清理7大系统内存区域，支持控制台、系统服务等多种运行方式。

✅ 优点

  • 内置三十余种语言，包含中文，新手可以快速上手
  • 配备全局快捷键、进程白名单、自定义托盘等实用功能
  • 支持定时、阈值双模式自动清理，可设置开机自启，运行优先级低
  • 程序经过官方签名认证，软件纯净无广告，安装方式灵活

❌ 缺点

  • 源码页面国内无法正常访问，用户不能自行编译修改程序
  • 功能设置项较多，整体界面看起来较为繁杂
  • 深度清理能力一般，高负载运行场景下内存释放效果有限

3. Reduce Memory

🔗 官网：https://www.sordum.org/9197/reduce-memory-v1-7/

工具简介

Sordum团队开发的经典轻量工具，主打极简一键清理，单文件免安装，专门解决程序内存泄漏、系统缓存堆积问题，对老旧Windows设备适配性良好。

✅ 优点

  • 极致轻量化设计，免安装、不写入注册表，便携性极强
  • 操作逻辑简单，无多余功能，打开即可直接使用
  • 兼容性出色，适配各类老旧系统，对低配老电脑优化效果明显
  • 完全免费使用，没有弹窗广告，退出后不会后台驻留

❌ 缺点

  • 仅支持手动清理，不具备定时自动清理功能
  • 只能清理基础进程内存，无法深度处理系统缓存
  • 整体优化能力有限，不适合高内存占用场景

✅ 优点

  • 全中文可视化界面，图形展示内存占用，上手毫无难度
  • 支持智能自动优化，可设置内存阈值，也能选择仅在CPU空闲时运行

  • 进阶设置选项较少，个性化自定义空间不足
  • 偶尔会弹出弹窗，推荐旗下其他相关软件

5. RAMMap

🔗 官网：https://docs.microsoft.com/en-us/sysinternals/downloads/rammap

工具简介

微软Sysinternals出品的专业内存分析工具，体积仅719KB，主打可视化解析内存分配数据，帮助用户定位内存占用异常、内存泄漏等问题，并非主打一键清理。

✅ 优点

  • 微软官方工具，安全可靠，无病毒、捆绑与误报问题
  • 功能专业全面，可查看物理内存、进程缓存、待机列表等各类数据
  • 支持内存快照保存与加载，方便排查系统各类疑难故障
  • 单文件即可运行，能够手动针对性释放指定区域内存

❌ 缺点

  • 没有自动定时清理功能，自动化使用体验较差
  • 无全局一键清理按钮，所有操作都需要手动完成
  • 纯英文界面，专业术语较多，学习门槛高，不适合纯小白

工具简介

老牌定时内存清理工具，分为免费版和付费专业版，兼容Windows 2000至Win10等老旧系统，默认依托系统计划任务定时清理，主打一次设置、长期使用。

❌ 缺点

  • 高级监控、多规则调度等功能需要付费解锁
  • 对Win11新版系统适配效果较差
  • 官网早已停止更新，后续没有版本维护
